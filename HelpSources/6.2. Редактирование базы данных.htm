Структура базы данных приложения

База данных создается динамически, данные в ней обновляются согласно подготовленным SQL-запросам. База, спроектированная для данной программы, проста в построениии, не имеет реляций, объединений, триггеров и т.д.

Данные хранятся в базе в том же виде, в каком и отображаются в программе, и задаются пользователем.

Для каждой библиотеки создается новая таблица в базе данных. Поля таблицы определяются пользовательской комбинацией из страндартных семи полей библиотек. Каждое поле библиотеки закреплено за своим типом в базе данных:

  • Поле "Строка" - тип VARCHAR(128)
  • Поле "Текст" - тип TEXT
  • Поле "Дата" - тип VARCHAR(20)
  • Поле "Дата + Время" - тип CHAR(20)
  • Поле "Оценка (5)" - тип CHAR(5)
  • Поле "Оценка (10)" - тип CHAR(10)
  • Поле "Приоритет" - тип VARCHAR(10)


В случае возникновения сбоя базы данных (вероятность этого ничтожно мала, но все же), ее можно будет открыть, отредактировать и исправить при помощи программного обеспечения DB Browser for SQLite.
